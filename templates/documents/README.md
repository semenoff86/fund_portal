# Шаблоны документов для Сервис-деска

Здесь хранятся шаблоны служебных записок и других внутренних документов МКК.

## Структура папок

```
templates/documents/
├── README.md           ← этот файл
├── source/             ← загрузите сюда исходные .docx шаблоны
├── metadata/           ← описание полей каждого шаблона (JSON)
└── generated/          ← сгенерированные документы (создаётся автоматически)
```

## Как добавить новый шаблон

### 1. Загрузите файл Word в `source/`

Используйте плейсхолдеры в формате Jinja2 (двойные фигурные скобки):

```
{{ full_name }}      — ФИО сотрудника
{{ department }}     — отдел
{{ document_date }}  — дата документа
{{ leave_start }}    — дата начала (пример для отпуска)
```

**Важно:** вставляйте плейсхолдер целиком в одном фрагменте текста Word (не разбивайте `{{` и `}}` на разные стили).

### 2. Создайте JSON в `metadata/`

Скопируйте `metadata/_example.json` и укажите:

| Поле | Описание |
|------|----------|
| `slug` | уникальный идентификатор (латиница, через дефис) |
| `title` | название в интерфейсе портала |
| `source_file` | путь к .docx относительно `templates/documents/` |
| `fields` | список полей формы |

### 3. Источники автозаполнения (`source` в поле)

| Значение | Откуда берётся |
|----------|----------------|
| `profile.full_name` | Личный кабинет → ФИО |
| `profile.department` | Личный кабинет → Отдел |
| `profile.email` | Личный кабинет → Email |
| `profile.phone` | Личный кабинет → Телефон |
| `profile.role` | Личный кабинет → Роль |
| `profile.username` | Логин |
| `auto.today` | Текущая дата |
| `auto.year` | Текущий год |
| *(не указан)* | Пользователь заполняет вручную в форме |

### 4. Перезапустите backend

После добавления файлов перезапустите API — шаблоны подхватываются из папки автоматически.
